# HTTP 메서드

## HTTP API를 만들어보자

#### 회원 정보 관리 API

1. URI 설계 - 리소스 식별
    - 행위는 리소스가 아니다.(등록, 조회, 수정, 삭제 등)
    - 회원이라는 개념 자체가 리소스다.
    - 계층 구조상 상위를 컬렉션으로 보고 복수 단어 사용 권장
      <br>
    - 회원 목록 조회: /members
    - 회원 조회, 등록, 수정, 삭제: /members/{id} → 행위 구분 필요
2. 리소스와 행위를 분리
    - URI는 리소스만 식별
    - 리소스와 해당 리소스를 대상으로 하는 행위를 분리
    - 리소스는 명사, 행위는 동사
